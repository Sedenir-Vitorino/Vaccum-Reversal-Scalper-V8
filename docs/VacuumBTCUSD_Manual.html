// 01 — Overview

What is Vacuum BTCUSD

A reversal scalper that identifies candle patterns and filters entries by trend direction and price position relative to support and resistance zones.

🕯️

Candle Patterns

Detects 7 classic patterns — Hammer, Inverted Hammer, Engulfing, Harami, Shooting Star. Filtered by candle body strength, ignores Dojis.

Active by default
📈

Trend Filter (EMA)

Defines market direction. Price above EMA → buys only. Price below → sells only. Avoids entries against the main trend.

Disabled by default
🎯

Support & Resistance

Calculates dynamic S/R from the last N candles. Integrated with EMA: buys only near support, sells only near resistance.

Disabled by default
📊

ATR Filter

Blocks entries when volatility is outside the normal range. Avoids explosive candles (news) and extreme ranging periods.

Active by default
🔒

Dynamic SL

When profit exceeds the configured floor, the Stop Loss rises automatically locking a growing share of the gain. Never drops.

Disabled by default
🛡️

SL/TP at Broker

SL and TP are always registered at the broker as absolute prices. If the code crashes or freezes, the order remains protected server-side.

Always active
// 02 — Parameters

All Parameters

Default values optimized by the backtester. Adjust freely — the defaults are the recommended starting point.

01

Volume

ParameterDefaultDescription
Volume0.01BTC amount per trade. For small accounts use 0.01 (minimum). 1 unit = 1 full BTC. Scale only after validating the strategy.
StopLossUSD$1.70Maximum loss per trade in dollars. SL is calculated as absolute price and sent directly to the broker. With 0.01 BTC equals ~170 price points.
TakeProfitUSD$0.20Profit target per trade in dollars. With 0.01 BTC equals ~20 price points. Easy to reach on BTCUSD M1 which moves 5-50 points per candle.
02

Dynamic SL

// Dynamic SL Formula
locked_value = FloorUSD + (current_profit × Percent%)
Example with defaults (Floor=$0.01, Percent=90.1%):
profit = $0.10 → locks $0.01 + 90.1%×$0.10 = $0.101
profit = $0.20 → locks $0.01 + 90.1%×$0.20 = $0.191
profit = $0.50 → locks $0.01 + 90.1%×$0.50 = $0.461
The SL never drops — only rises as profit increases.
ParameterDefaultDescription
UseSLDinamicofalseEnables Dynamic SL. When active, the SL rises automatically as profit grows. Recommended to enable after validating the robot with fixed SL.
SLDinamicoPisoUSD$0.01Minimum guaranteed value that the SL always locks. Works as a minimum breakeven — even if the percentage adds little, the floor guarantees this gain.
SLDinamicoPercent90.1%Percentage of current profit added to the floor. With 90.1%, if profit=$0.20, SL locks $0.191. Higher % = more aggressive trailing. Range: 1% to 99%.
03

Candle Patterns

ParameterDefaultDescription
MinCandleStrength28.9%Minimum candle strength — ratio between body and total range. 28.9% means the body must be at least 28.9% of the total candle size. Filters weak and inconclusive candles.
IgnoreDojitrueIgnores Doji candles (body less than 5% of range) as entry signal. Dojis are ambiguous and generate many false positives on M1.
UseHammerfalseHammer — candle with long lower shadow (≥2× body) and short upper shadow. Bearish to bullish reversal signal.
UseInvertedHammertrueInverted Hammer — long upper shadow (≥2× body), bullish body. Bullish reversal signal after a decline.
UseBullishEngulfingfalseBullish Engulfing — bullish candle that completely engulfs the previous bearish candle. One of the most reliable bullish reversal patterns.
UseBullishHaramifalseBullish Harami — small bullish candle inside the body of the previous bearish candle. Indicates hesitation and possible reversal.
UseShootingStarfalseShooting Star — long upper shadow, small body at top. Bearish signal of rejection of higher prices.
UseBearishEngulfingtrueBearish Engulfing — bearish candle that engulfs the previous bullish candle. Strong bearish reversal signal, especially at resistances.
UseBearishHaramifalseBearish Harami — small bearish candle inside the body of the previous bullish candle. Softer signal of possible reversal.
04

Trend Filter (EMA)

💡

How EMA filters entries

With UseEMA active, the robot only opens buys when price is above the EMA (uptrend) and only opens sells when price is below (downtrend). This avoids entries against the main market movement.

ParameterDefaultDescription
UseEMAfalseActivates the trend filter via EMA. When on, only trades in the direction confirmed by the moving average. Recommended to activate to reduce losses in sideways markets.
EMAPeriod165Period of the EMA. 165 periods on M1 = ~2h45min of history. Shorter EMAs (20-50) are more reactive; longer ones (100-200) filter more but miss more entries.

UseSRfalseActivates the S/R filter. When on together with EMA, entries only occur at the right point: buys in support zone in uptrend, sells in resistance zone in downtrend.
CandlesSR183Number of candles to calculate dynamic S/R. 183 candles on M1 = ~3 hours. The robot uses the highest high and lowest low of this period as resistance and support.

06

ATR Filter (Volatility)

ParameterDefaultDescription
UseATRtrueActivates the volatility filter. Blocks entries when current ATR is too far from the average — avoids explosive news candles and extreme ranging periods.
ATRPeriod8Fast ATR period. Slow ATR uses ATRPeriod × 3 = 24 candles. The comparison between them determines if current volatility is "normal" for the asset.
ATRMaxMult1.3×Maximum multiplier: fast ATR cannot be greater than 1.3× the slow ATR. Blocks entries during extreme volatility (news, liquidations).
ATRMinMult0.3×Minimum multiplier: fast ATR cannot be less than 0.3× the slow ATR. Blocks entries when the market is completely dead and without movement.
07

Daily Protection

⚠️

When daily protection is activated

When any of the limits below are reached, the robot stops opening new trades until midnight UTC the next day. Open positions continue to be managed normally.

ParameterDefaultDescription
MaxTradesPerDay61Maximum number of trades per day. At 61 trades, the robot stops and waits for the next day. Avoids overtrading on days with many false signals.
MaxDailyLossUSD$1.20Maximum accumulated daily loss. When losing $1.20 total for the day (sum of all trades), the robot stops immediately. Protects capital from bad days.
MaxConsecLosses3Maximum consecutive losses. After 3 losses in a row without any win, the robot stops. Indicates the market may be in a bad mode for the strategy.
CooldownMinutes2 minMinimum interval between trades. The robot waits 2 minutes after closing a position before opening the next. Avoids burst entries at the same price level.
CloseOnSLTPFailuretrueIf after 5 attempts the SL/TP cannot be applied to the broker, the robot closes the position for safety. Never leaves an order exposed without protection.
08

Display

ParameterDefaultDescription
ShowPaneltrueDisplays the information panel in the top-right corner of the chart. Shows account status, open position, active filters and daily counters in real time.
ShowDebugtrueDisplays detailed logs in the cTrader console. Shows each entry block with the reason (EMA, S/R, ATR), SL/TP calculations and confirmations. Disable in production for better performance.
// 03 — Logic

Operation Flow

How the bot decides to open, manage and close each position.

1

Daily Check

Every candle, the bot checks if daily limits have been reached (max loss, trade count, consecutive losses, cooldown). If any limit is active, no new trades are opened.

2

Candle Pattern Detection

On the previous closed candle (index -2), the bot checks if any active pattern is present — Inverted Hammer, Bearish Engulfing, etc. Candle strength must be ≥ MinCandleStrength.

3

Trend Filter (EMA)

If UseEMA is active: bullish patterns are only valid with price above the EMA. Bearish patterns only valid below. Without EMA, the bot trades both directions freely.

4

S/R Filter

If UseSR is active: buys only occur when price is in the support zone (lower part of the range). Sells only in the resistance zone (upper part). Works together with the EMA — not independently.

5

ATR Filter

The fast ATR is compared to the slow ATR. If volatility is too high (>1.3×) or too low (<0.3×), the entry is blocked to avoid abnormal market conditions.

6

Order Opening

The order is opened WITHOUT SL/TP initially (always accepted by broker). Then SL and TP are applied as ABSOLUTE PRICES via ModifyPosition with up to 5 attempts. If it fails, the order is closed for safety.

7

Real-Time Management (OnTick)

Every tick: checks if SL/TP are still present (reapplies if missing), updates peak profit, and applies Dynamic SL if active and profit ≥ configured floor.

8

Closing

The position closes when price touches the TP (profit) or SL (loss) registered at the broker. The code does not close manually except in case of SL/TP failure or robot freeze.

// 05 — Installation

How to Install

1

Open the Editor in cTrader

In cTrader, go to Automate → New cBot. Name it VacuumBTCUSD.

2

Paste the Code

Replace all editor content with the code from the file VacuumBTCUSD.cs. Press F6 to compile.

3

Set Up the Chart

Open a BTCUSD M1 chart. Drag the cBot to the chart or double-click in the compiled bots list.

4

Load Parameters (optional)

In the parameters window, click the gear icon ⚙ and select Import Parameters. Import the optimized JSON file.

5

Test on Demo Account

Always run on a demo account first for at least 1-2 days. Monitor the log with ShowDebug: true to confirm SL/TP are applied correctly.

✅

SL/TP Confirmation in Log

After opening each order, the log should show: SL/TP aplicados (tentativa 1) ✓ | SL=84360.00 TP=84550.00. This confirms protection was registered at the broker.

⚠️

Warning: BTCUSD only

This bot is calibrated exclusively for BTCUSD CFD. The USD→price conversion formula uses this asset's specific characteristics (LotSize=1, PipSize=1.0). Using on other assets will produce incorrect SL/TP values.
